📦 Section 1 — Fundamentals Q1–20 ▾
Q1 What is Node.js? ▶

Node.js is an open-source, cross-platform JavaScript runtime built on Chrome's V8 engine. It allows JavaScript to run on the server side (outside the browser).

Q2 What is the V8 Engine? ▶

V8 is Google's open-source JavaScript engine written in C++. It compiles JS directly to machine code, making execution very fast. Node.js uses V8 to execute JS on the server.

Q3 What makes Node.js different from other server-side technologies like PHP or Java? ▶

Q4 What is the Event Loop in Node.js? ▶

The Event Loop is the core mechanism that allows Node.js to perform non-blocking I/O despite being single-threaded. It offloads operations to the OS/libuv thread pool, and when they complete, the callbacks are pushed to the call stack.

Phases of the Event Loop:

  1. timers — executes setTimeout and setInterval callbacks
  2. pending callbacks — I/O callbacks deferred to next loop
  3. idle, prepare — internal use
  4. poll — retrieve new I/O events
  5. check — setImmediate callbacks
  6. close callbacks — e.g., socket.on('close')
Q5 Is Node.js single-threaded? ▶

Yes, Node.js runs JavaScript on a single main thread. However, I/O operations are handled by libuv's thread pool (4 threads by default), so the main thread is never blocked.

Q6 What is non-blocking I/O? ▶

Non-blocking I/O means that when a file read or network request is made, Node.js does not wait for it to complete. It registers a callback and moves to the next task.

const fs = require('fs');

// Non-blocking
fs.readFile('file.txt', 'utf8', (err, data) => {
  console.log(data); // runs later
});

console.log('This runs first'); // runs immediately

Q8 What is setTimeout(() => {}, 0) vs setImmediate()? ▶

Both execute after I/O, but setTimeout with 0ms delay can fire before or after setImmediate depending on the context. Inside an I/O callback, setImmediate always fires first.

Q9 What are streams in Node.js? ▶

Streams are objects that let you read/write data piece by piece (chunks) rather than loading everything into memory at once.

Types of streams:

  • Readable — reading data (e.g., fs.createReadStream)
  • Writable — writing data (e.g., fs.createWriteStream)
  • Duplex — both read and write (e.g., TCP socket)
  • Transform — duplex + transforms data (e.g., zlib.createGzip)
Q10 What is the pipe() method in streams? ▶

pipe() connects a Readable stream to a Writable stream, automatically handling backpressure.

const fs = require('fs');
const readable = fs.createReadStream('input.txt');
const writable = fs.createWriteStream('output.txt');
readable.pipe(writable);

Q12 What is module.exports vs exports? ▶

module.exports is the actual object returned by require(). exports is a shorthand reference to module.exports.

// Both work for adding properties
exports.greet = () => 'Hello';

Q13 What is __dirname and __filename in Node.js? ▶

__dirname — absolute path of the directory containing the current file.
__filename — absolute path of the current file.

console.log(__dirname);  // /home/zaheer/project
console.log(__filename); // /home/zaheer/project/app.js
Q14 What is the path module? ▶

A built-in Node.js module to work with file and directory paths in an OS-independent way.

const path = require('path');
path.join('/home', 'zaheer', 'app.js');   // /home/zaheer/app.js
path.resolve('src', 'index.js');           // absolute path
path.basename('/home/zaheer/app.js');      // app.js
path.extname('app.js');                    // .js
path.dirname('/home/zaheer/app.js');       // /home/zaheer
Q15 What is the fs module? ▶

The fs (File System) module provides APIs to interact with the filesystem.

const fs = require('fs');

// Async read
fs.readFile('file.txt', 'utf8', (err, data) => { ... });

// Sync read (blocks event loop — avoid in production)
const data = fs.readFileSync('file.txt', 'utf8');

// Write
fs.writeFile('out.txt', 'Hello', (err) => { ... });

Q19 What are Buffers in Node.js? ▶

Buffers represent fixed-length raw binary data. Used for handling binary data like files, network packets, etc.

const buf = Buffer.from('Hello', 'utf8');

console.log(buf.toString()); // Hello

  • npm install <pkg> --save-dev — installs to devDependencies (only during development)
⚡ Section 2 — Async Patterns Q21–40 ▾
Q21What is a callback in Node.js?▶

A callback is a function passed as an argument to another function, invoked after an async operation completes.

function fetchData(callback) {
  setTimeout(() => {
    callback(null, { name: 'Zaheer' });
  }, 1000);
}

fetchData((err, data) => {
  if (err) return console.error(err);
  console.log(data);
});
Q22What is "Callback Hell"?▶

Deeply nested callbacks that make code hard to read and maintain.

getUser(id, (err, user) => {
  getPosts(user.id, (err, posts) => {
    getComments(posts[0].id, (err, comments) => {
      // deeply nested...
    });
  });
});

Solution: Use Promises or async/await.

Q23What is a Promise?▶

A Promise represents the eventual completion (or failure) of an async operation. States: pending, fulfilled, rejected.

const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve('Done!'), 1000);
});

promise
  .then(result => console.log(result))  // Done!
  .catch(err => console.error(err));
Q24What is async/await?▶

async/await is syntactic sugar over Promises that makes async code look synchronous.

async function fetchUser() {
  try {
    const user = await getUser(1);
    const posts = await getPosts(user.id);
    return posts;
  } catch (err) {
    console.error(err);
  }
}
Q25What is Promise.all()?▶

Runs multiple promises in parallel and resolves when all succeed, or rejects if any one fails.

const [users, posts] = await Promise.all([
  fetchUsers(),
  fetchPosts()
]);
Q26What is Promise.allSettled()?▶

Like Promise.all() but never rejects — waits for all promises and returns their statuses.

const results = await Promise.allSettled([p1, p2, p3]);
results.forEach(r => console.log(r.status, r.value || r.reason));
Q27What is Promise.race()?▶

Resolves or rejects as soon as the first promise settles.

const result = await Promise.race([
  fetch('/api/fast'),
  timeout(5000)   // reject after 5 sec
]);
Q28What is Promise.any()?▶

Resolves with the first fulfilled promise. Rejects only if all promises reject.

Q30How do you handle errors in async/await?▶ 
// Option 1: try/catch
async function run() {
  try {
    const data = await riskyOperation();
  } catch (err) {
    console.error('Error:', err.message);
  }
}

// Option 2: .catch on the call
run().catch(console.error);
Q31Difference between synchronous and asynchronous code?▶

Synchronous: Executes line by line, each line waits for the previous to complete. Blocks the thread.

Asynchronous: Operations are started and the program continues. Result is handled via callback/promise/async-await.

Q32What is util.promisify()?▶

Converts a callback-based function into a Promise-based one.

const util = require('util');
const fs = require('fs');

const readFile = util.promisify(fs.readFile);
const data = await readFile('file.txt', 'utf8');
Q33What are microtasks and macrotasks?▶

Microtasks (higher priority): Promise callbacks (.then), process.nextTick, queueMicrotask

Macrotasks (lower priority): setTimeout, setInterval, setImmediate, I/O callbacks

Microtasks run before macrotasks after each task.

Q34Explain the output of this code:▶ 
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');

Output: 1 → 4 → 3 → 2

  • 1, 4 are synchronous
  • Promise .then is a microtask (runs before macrotasks)
  • setTimeout is a macrotask (runs last)
Q35Difference between setInterval and setTimeout?▶

setTimeout(fn, ms) — runs fn once after ms milliseconds.

setInterval(fn, ms) — runs fn repeatedly every ms milliseconds.

const timer = setInterval(() => console.log('tick'), 1000);
clearInterval(timer); // stop it
Q36What is an IIFE? Is it useful in Node.js?▶

An Immediately Invoked Function Expression executes as soon as it's defined. Every CommonJS module is wrapped in an IIFE-like wrapper by Node.js.

(function() {
  const x = 10; // scoped, not global
})();
Q37What is async_hooks module?▶

async_hooks is a Node.js core module to track the lifetime of async resources (for APM/tracing tools). Not commonly used in day-to-day development.

Q38Can you await outside an async function?▶

In Node.js v14.8+ with ES Modules ("type": "module"), you can use top-level await:

// works in .mjs or type:module
const data = await fetch('https://api.example.com');
Q39What is the Worker Threads module?▶

Allows running JavaScript in parallel threads for CPU-intensive tasks. Each worker has its own V8 instance.

const { Worker } = require('worker_threads');
const worker = new Worker('./heavy-task.js');
worker.on('message', (result) => console.log(result));
Q40What is child_process module?▶

Allows spawning child processes to run shell commands or other scripts.

const { exec } = require('child_process');
exec('ls -la', (err, stdout) => {
  console.log(stdout);
});

Q53How do you create a custom module in Node.js?▶

Q54What is circular dependency in Node.js?▶

When module A requires B, and B requires A. Node.js handles it by returning a partially-filled exports object, which can lead to undefined values. Should be avoided by refactoring.

🔐 Section 5 — Security & Performance Q71–85 ▾
Q71What is JWT (JSON Web Token)?▶

JWT is a compact, URL-safe token format for authentication. It contains a header.payload.signature.

const jwt = require('jsonwebtoken');

// Sign
const token = jwt.sign({ userId: 1 }, process.env.JWT_SECRET, { expiresIn: '7d' });

// Verify
const decoded = jwt.verify(token, process.env.JWT_SECRET);
Q72What is bcrypt and why is it used?▶

bcrypt is used to hash passwords so they're never stored in plain text. It includes salt to prevent rainbow table attacks.

const bcrypt = require('bcryptjs');
const hash = await bcrypt.hash('myPassword', 10);   // 10 = salt rounds
const match = await bcrypt.compare('myPassword', hash); // true
Q74What is Helmet.js?▶

helmet is an Express middleware that sets security-related HTTP headers to protect against common attacks.

const helmet = require('helmet');
app.use(helmet());
// Sets: Content-Security-Policy, X-XSS-Protection, etc.
Q78How do you handle file uploads in Node.js?▶

Q81What is PM2?▶

PM2 is a production process manager for Node.js that handles auto-restart on crash, cluster mode, log management, and zero-downtime reload.

pm2 start index.js --name my-app

Q84How do you handle uncaught exceptions in Node.js?▶

🍃 Section 7 — MongoDB Basics Q1–20 ▾
Q1What is MongoDB?▶

MongoDB is a NoSQL document database that stores data as flexible JSON-like documents (BSON format). It's schema-less, highly scalable, and ideal for applications with dynamic or evolving data structures.

Q2Difference between SQL and NoSQL databases?▶
SQL (MySQL)NoSQL (MongoDB)
Data modelTables & rowsDocuments, collections
SchemaFixed schemaFlexible/schema-less
RelationshipsJOINsEmbedded docs / $lookup
ScalabilityVerticalHorizontal
Best forComplex relationsLarge, flexible, fast I/O
Q3What is BSON?▶

BSON (Binary JSON) is MongoDB's internal storage format — a binary-encoded serialization of JSON-like documents. It supports additional types like Date, ObjectId, Binary.

Q6What is _id in MongoDB?▶

_id is the primary key of every document. If not provided, MongoDB auto-generates an ObjectId — a 12-byte unique identifier.

ObjectId("507f1f77bcf86cd799439011")
// 4 bytes timestamp + 5 bytes random + 3 bytes increment
Q7What is the MongoDB CRUD syntax?▶
// Create
db.users.insertOne({ name: 'Zaheer' });
db.users.insertMany([{ name: 'A' }, { name: 'B' }]);

// Read
db.users.find({ name: 'Zaheer' });
db.users.findOne({ _id: id });

// Update
db.users.updateOne({ name: 'Zaheer' }, { $set: { age: 22 } });
db.users.updateMany({ role: 'user' }, { $set: { active: true } });

// Delete
db.users.deleteOne({ name: 'Zaheer' });
db.users.deleteMany({ active: false });
Q8What are common MongoDB query operators?▶
$eq, $ne, $gt, $gte, $lt, $lte  // comparison
$in, $nin                         // in array
$and, $or, $not, $nor             // logical
$exists, $type                    // element
$regex                            // pattern match

db.users.find({ age: { $gte: 18, $lte: 30 } });
db.users.find({ role: { $in: ['admin', 'moderator'] } });
Q9What are MongoDB update operators?▶
$set      // set field value
$unset    // remove a field
$inc      // increment numeric field
$push     // add item to array
$pull     // remove item from array
$addToSet // add to array only if not present
$rename   // rename a field
Q10What is an index in MongoDB? Why is it important?▶

Indexes are data structures that speed up query performance by avoiding full collection scans.

db.users.createIndex({ email: 1 });         // ascending
db.users.createIndex({ email: 1 }, { unique: true }); // unique
db.users.createIndex({ name: 'text' });     // text search

Without indexes, MongoDB scans every document — slow for large collections.

Q12What is Aggregation in MongoDB?▶

Aggregation processes data through a pipeline of stages to transform, filter, group, or compute data.

db.orders.aggregate([
  { $match: { status: 'completed' } },
  { $group: { _id: '$userId', total: { $sum: '$amount' } } },
  { $sort: { total: -1 } },
  { $limit: 10 }
]);
Q13What is $lookup in MongoDB?▶

$lookup performs a left outer join between two collections (like SQL JOIN).

db.orders.aggregate([
  {
    $lookup: {
      from: 'users',
      localField: 'userId',
      foreignField: '_id',
      as: 'user'
    }
  }
]);
Q14Difference between embedded documents and references?▶
EmbeddedReference
StorageNested inside documentSeparate collection, store ObjectId
ReadOne queryMultiple queries or $lookup

Q19What is a Replica Set?▶

A replica set is a group of MongoDB servers that maintain the same dataset for high availability. One is the primary (receives writes), others are secondaries (replicate data). If primary fails, a secondary is elected automatically.

🔗 Section 8 — Mongoose Q21–40 ▾
Q21What is Mongoose?▶

Mongoose is an ODM (Object Data Modeling) library for MongoDB and Node.js. It provides schema-based modeling, validation, query building, and middleware for MongoDB documents.

Q23What is a Mongoose Schema?▶

A Schema defines the structure, types, validations, and defaults for a MongoDB collection.

const { Schema, model } = require('mongoose');

const userSchema = new Schema({
  name:  { type: String, required: true, trim: true },
  email: { type: String, required: true, unique: true, lowercase: true },
  age:   { type: Number, min: 0, max: 120 },
  role:  { type: String, enum: ['user', 'admin'], default: 'user' },
  createdAt: { type: Date, default: Date.now }
});

const User = model('User', userSchema);
module.exports = User;
Q26What is CRUD with Mongoose?▶
// CREATE
const user = await User.create({ name: 'Zaheer', email: 'z@z.com' });

// READ
const users = await User.find({});
const user  = await User.findById(id);
const user  = await User.findOne({ email: 'z@z.com' });

// UPDATE
await User.findByIdAndUpdate(id, { name: 'New' }, { new: true });
await User.updateMany({ role: 'user' }, { $set: { active: true } });

// DELETE
await User.findByIdAndDelete(id);
await User.deleteMany({ active: false });
Q27What does { new: true } mean in findByIdAndUpdate?▶

By default, findByIdAndUpdate returns the old document before the update. { new: true } makes it return the updated document.

Q29What is Mongoose Middleware (Hooks)?▶

Functions that run before or after certain operations.

// Pre-save: hash password before saving
userSchema.pre('save', async function(next) {
  if (!this.isModified('password')) return next();
  this.password = await bcrypt.hash(this.password, 10);
  next();
});

// Post-save

  console.log('User saved:', doc.email);
});

Q4What is middleware in Express?▶

Middleware functions have access to req, res, and next. They can execute code, modify req/res, end the cycle, or call next().

app.use((req, res, next) => {

  next(); // MUST call next() or request hangs
});

Q22How do you build authentication in Express with JWT?▶
// Login
app.post('/login', async (req, res) => {
  const { email, password } = req.body;
  const user = await User.findOne({ email });
  if (!user || !(await bcrypt.compare(password, user.password)))
    return res.status(401).json({ error: 'Invalid credentials' });

  const token = jwt.sign({ id: user._id }, process.env.JWT_SECRET, { expiresIn: '7d' });
  res.json({ token });
});

// Auth middleware
const authenticate = async (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'No token' });
  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET);
    next();
  } catch {
    res.status(401).json({ error: 'Invalid token' });
  }
};
