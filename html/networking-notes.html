1. How Websites Work

From typing a URL to displaying pixels on the screen.

When you type a URL like google.com into your browser and press Enter, a sophisticated sequence of networked requests and file parsing initiates almost instantaneously.

The Step-by-Step Lifecycle

  • DNS Resolution: The browser queries Domain Name System (DNS) servers to translate the human-readable domain (google.com) into an IP address (e.g., 142.250.190.46).
  • TCP Handshake: The browser establishes a reliable Transmission Control Protocol (TCP) connection with the destination server via a 3-way handshake (SYN, SYN-ACK, ACK).
  • TLS/SSL Handshake: If encrypted via HTTPS, cryptographic keys are exchanged to secure all subsequent communications.
  • HTTP Request: The client dispatches an HTTP GET request asking for the root document (`index.html`).
  • Server Response: The backend server computes the request, retrieves assets, and replies with standard HTTP response headers and the payload body.
  • Browser Parsing: The client parses HTML into the DOM tree, CSS into the CSSOM tree, executes JavaScript, and paints the visible page layout.
💡
DNS Analogy: Think of DNS as the absolute phonebook of the internet. Humans remember names, but routing machines communicate exclusively via raw numerical IP addresses.
System Models

2. Client vs Server

The standard multi-tier distributed computing model.

💻 The Client (Frontend)

  • Runs locally on the end-user's consumer hardware (Browser, Mobile App).
  • Responsible for user interfaces, dynamic visual layout, and capturing events.
  • Parses documents, computes presentation logic, and renders output.
  • Highly untrusted execution environment; variables can be inspected or manipulated.

🗄️ The Server (Backend)

  • Runs remotely in highly optimized cloud datacenters.
  • Responsible for authenticating users, central logic, and database persistence.
  • Serves requested API endpoints, static assets, and dynamic payloads.
  • Highly secure, trusted environment managing sensitive credentials.
Protocols

3. HTTP vs HTTPS

Securing data transport across the public internet.

Feature HTTP (Hypertext Transfer Protocol) HTTPS (HTTP Secure)
Encryption None. Transmits plain text payloads openly. Encrypted symmetrically using TLS/SSL protocols.
Default Port Port 80 Port 443
Security Vulnerable to packet sniffing and Man-in-the-Middle (MitM). Highly resilient; validates site authenticity via CA Certificates.
Performance Slightly faster due to skipping initial cryptographic handshakes. Enables HTTP/2 and HTTP/3 multiplexing natively for superior loads.
🔒
TLS Handshake Standard: Modern HTTPS leverages asymmetric cryptography (Public/Private keys) to exchange symmetric session keys safely, providing optimal performance for streaming massive payloads.

4. APIs (Application Programming Interfaces)

How decoupled software applications talk to each other.

An API is a contractual set of rules defines how independent software entities interact. In web development, REST APIs and GraphQL APIs serve as universal bridges passing serialized data back and forth.

HTTP Request Lifecycle 
GET /api/v1/users/42 HTTP/1.1
Host: api.devhub.internal
Authorization: Bearer eyJhbGciOiJIUzI1Ni...
Accept: application/json

── Server responds with HTTP/1.1 200 OK ──
📐
REST API
Resource-based routing utilizing standard HTTP verbs (GET, POST, PUT, DELETE). Stateless architecture.
🕸️
GraphQL
Query-based architecture allowing clients to fetch precisely the structured payload shapes they require.
⚡
WebSockets
Persistent full-duplex TCP communication channels ideal for live chats and stock price streaming.

5. JSON (JavaScript Object Notation)

The universal standard for lightweight data interchange.

Derived from JavaScript syntax, JSON is language-agnostic. All property keys must be wrapped in double quotes. It supports basic scalar primitives (strings, numbers, booleans, null) along with ordered arrays and nested objects.

Engine Internals

6. Browser Rendering Pipeline

How web engines convert raw byte arrays into painted interactive screens.

Critical Rendering Path

  1. DOM Construction: Bytes are transformed into Characters -> Tokens -> Nodes -> DOM tree.
  2. CSSOM Construction: Concurrent CSS byte parsing produces the CSS Object Model tree.
  3. Render Tree Assembly: DOM and CSSOM combine, omitting non-visual hidden elements (`display: none`).
  4. Layout (Reflow): Engine calculates exact geometric positioning and sizing dimensions for each node.
  5. Paint: Converts node layout boundaries into concrete raster pixels across rendering layers.
  6. Compositing: Renders visual layers overlapping correctly onto the physical screen buffer.

8. Package Managers: npm vs pnpm

Managing third-party open source code dependencies efficiently.

Metric npm (Node Package Manager) pnpm (Performant npm)
Storage Architecture Flat file structure inside node_modules. Duplicates identical packages across distinct projects. Centralized global content-addressable store. Hard links dependencies directly into isolated local maps.
Disk Efficiency Extremely heavy disk space footprint. Saves gigabytes of storage space via unique immutable pointers.
Installation Speed Standard synchronous sequential resolution pipelines. Blazing fast concurrent downloads and symlink layouts.
Strictness Allows accessing phantom hoisted dependencies openly. Strict isolated layouts prevent ghost imports natively.

12. Environment Variables

Decoupling dynamic system parameters from permanent codebase logic.

Environment Variables define configurable strings injected into execution scopes dynamically. This securely decouples API keys, database URLs, and port mappings from public git repositories.

.env Schema Mapping 
# Production Environment Maps
PORT=8080
NODE_ENV=production
DATABASE_URL="postgresql://user:secretpass@db.internal:5432/prod"
JWT_SECRET_KEY="super-secret-cryptographic-hash-salt-phrase"
⚠️
Security Mandate: Always append .env inside your root .gitignore configuration file instantly. Exposing database credentials or access tokens publicly leads to severe infrastructural compromises.
