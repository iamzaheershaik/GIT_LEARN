📄 Introduction

What is HTML5?

HTML5 is the 5th and current version of HTML — the standard language for creating web pages. Released in 2014 by W3C, it introduced semantic elements, native audio/video, form improvements, canvas, local storage, and much more.

🏗️
Structure
Defines the skeleton of every webpage — headings, paragraphs, sections, links.
🧠
Semantic Meaning
New tags like <article>, <nav>, <main> describe content meaning.
🎯
Accessibility
ARIA roles and semantic HTML make sites usable for everyone, including screen readers.
🔍
SEO Friendly
Search engines read HTML to understand page content. Proper markup boosts rankings.

Basic HTML5 Document Structure

HTML5 Boilerplate 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Page description for SEO">
  <title>Page Title</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header>...</header>
  <main>
    <section>...</section>
    <article>...</article>
  </main>
  <footer>...</footer>
  <script src="app.js"></script>
</body>
</html>
💡
DOCTYPE: Always the very first line. <!DOCTYPE html> tells the browser to render in HTML5 standards mode — never omit it.
🏷️ Semantic Tags

Semantic HTML5 Tags

Semantic tags tell the browser (and search engines) what a piece of content means — not just how it looks. They replaced the old soup of <div id="nav"> patterns.

❌ Non-semantic (Old way)

  • All <div> and <span>
  • No meaning to the browser
  • Bad for accessibility
  • Harder for SEO crawlers

✅ Semantic (HTML5 way)

  • Meaningful tags describe content
  • Screen readers understand structure
  • Better SEO ranking
  • Cleaner, maintainable code

Page Layout Tags

TagPurposeUsage Tip
<header>Intro/top of page or section. Can contain logo, nav, h1.One per page typically, or per <article>
<nav>Navigation links block (menus, breadcrumbs).Wrap only major nav, not all links
<main>The dominant content of the page. Only ONE per page.Never put inside <header> or <footer>
<section>Thematic grouping. Should have a heading inside.Use when content is related but not standalone
<article>Self-contained, reusable content (blog post, card, product).Should make sense on its own, out of context
<aside>Tangentially related content — sidebars, callout boxes.Related to nearby content but could be removed
<footer>Closing content — copyright, links, contact info.Can be used inside <article> too
<address>Contact info for nearest article or body element.Not just postal — can be email/phone/social

Content Tags

TagMeaningBlock/Inline
<h1>–<h6>Headings. h1 = most important. Use only ONE h1 per page.Block
<p>Paragraph of text.Block
<strong>Important/bold text. Has semantic weight (not just visual).Inline
<em>Emphasis/italic. Stress emphasis on a word.Inline
<mark>Highlighted/relevant text (search result highlight).Inline
<time>Machine-readable date/time. Use datetime attribute.Inline
<abbr>Abbreviation. Use title attr for full expansion.Inline
<cite>Title of a work being referenced.Inline
<code>Inline code snippet.Inline
<pre>Preformatted text — whitespace preserved.Block
<blockquote>Long quotation from another source. Use cite attr.Block
<figure> + <figcaption>Self-contained media with caption (image, chart, code).Block
<details> + <summary>Native accordion/disclosure widget. No JS needed.Block
<dialog>Native modal dialog element. Use JS .showModal().Block

Full Page Structure Example

Semantic Layout 
<body>
  <header>
    <nav>
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h1>Blog Post Title</h1>
        <time datetime="2025-05-12">May 12, 2025</time>
      </header>
      <section>
        <h2>Introduction</h2>
        <p>Content goes here...</p>
      </section>
      <figure>
        <img src="chart.png" alt="Sales chart">
        <figcaption>Q1 2025 Sales Data</figcaption>
      </figure>
    </article>
    <aside>
      <h3>Related Posts</h3>
    </aside>
  </main>

  <footer>
    <address>Contact: <a href="mailto:hello@example.com">hello@example.com</a></address>
  </footer>
</body>
✅
Rule of thumb: Use <div> only when no semantic tag fits. If it's navigation → <nav>. If it's a blog post → <article>. If it's a side column → <aside>.
📝 Forms & Inputs

Forms & Input Elements

Forms are how users send data to your server. HTML5 added a ton of new input types and attributes that give you built-in validation for free.

Form Structure

Basic Form 
<form
  action="/submit"       <!-- Where to send data -->
  method="POST"        <!-- GET or POST -->
  enctype="multipart/form-data"  <!-- needed for file uploads -->
  novalidate           <!-- disables browser validation -->
>
  <fieldset>
    <legend>Personal Info</legend>
    <label for="name">Full Name</label>
    <input
      type="text"
      id="name"
      name="name"
      placeholder="Enter your name"
      required
      autocomplete="name"
    >
  </fieldset>
  <button type="submit">Submit</button>
</form>

Input Attributes (HTML5)

required
Field must be filled before submit. Browser shows error automatically.
placeholder
Hint text shown inside empty field. Not a substitute for <label>.
pattern
Regex validation. Example: pattern="[A-Za-z]{3}"
min / max
Min/max value for number, date, range inputs.
minlength / maxlength
Character length limits for text inputs.
step
Increment size for number/range inputs. Use step="0.01" for decimals.
disabled
Field is not interactive. Value is NOT sent on submit.
readonly
User can't change value but it IS sent on submit.
autofocus
Automatically focused when page loads. Use sparingly (one per page).
autocomplete
Hints browser for autofill: "name", "email", "off".
multiple
Allows multiple values. For file and email inputs.
accept
Filter file types: accept=".pdf,.jpg,image/*"
list
Links to a <datalist> id for autocomplete suggestions.
form
Associate input with a form by its id, even if outside the form tag.

Other Form Elements

select, textarea, datalist, output 
<!-- Dropdown Select -->
<select name="city" multiple>
  <optgroup label="Maharashtra">
    <option value="pune">Pune</option>
    <option value="mumbai" selected>Mumbai</option>
  </optgroup>
</select>

<!-- Textarea -->
<textarea
  name="bio"
  rows="5"
  cols="40"
  maxlength="500"
  placeholder="Tell us about yourself..."
></textarea>

<!-- Datalist: autocomplete suggestions -->
<input type="text" list="frameworks" placeholder="Pick a framework">
<datalist id="frameworks">
  <option value="React">
  <option value="Vue">
  <option value="Svelte">
</datalist>

<!-- Output: display calculated result -->
<input type="range" id="a" value="50">
<output for="a">50</output>
⚠️
Always use labels! Every input needs an associated <label for="id">. It improves accessibility, clickable hit area, and screen reader support. Never use placeholder as a substitute.
📊 Tables

HTML Tables

Tables are for tabular data — rows and columns with a relationship. Never use them for layout purposes (that's CSS Grid's job).

Complete Table Anatomy

Full Table Structure 
<table>
  <caption>Monthly Sales Report 2025</caption> <!-- visible title -->

  <colgroup>
    <col span="1" style="background:#f9f9f9">
    <col span="3">
  </colgroup>

  <thead>                          <!-- header rows -->
    <tr>
      <th scope="col">Month</th>      <!-- scope for a11y -->
      <th scope="col">Units</th>
      <th scope="col">Revenue</th>
      <th scope="col">Growth</th>
    </tr>
  </thead>

  <tbody>                          <!-- data rows -->
    <tr>
      <th scope="row">January</th>   <!-- row header -->
      <td>120</td>
      <td>₹1,20,000</td>
      <td>+12%</td>
    </tr>
    <tr>
      <th scope="row">February</th>
      <td colspan="2">No Data</td>  <!-- spans 2 cols -->
      <td>-</td>
    </tr>
  </tbody>

  <tfoot>                          <!-- summary row -->
    <tr>
      <th scope="row">Total</th>
      <td>120</td>
      <td>₹1,20,000</td>
      <td rowspan="1">+12%</td>
    </tr>
  </tfoot>
</table>

Key Table Attributes

AttributeElementPurpose
scope="col"<th>Header applies to entire column (required for accessibility)
scope="row"<th>Header applies to entire row
colspan="N"<td>, <th>Cell spans N columns horizontally
rowspan="N"<td>, <th>Cell spans N rows vertically
headers="id"<td>Links cell to header by id — complex tables
id<th>Used with headers attribute for complex tables
🚫
Never use tables for layout! That was a 1990s practice. Use CSS Grid or Flexbox instead. Tables are only for data that has row + column relationships.
♿ Accessibility (a11y)

Accessibility (a11y)

Accessibility means building websites that everyone can use — including people with visual, motor, auditory, or cognitive disabilities. It's a legal requirement in many countries and good practice always.

👁️
Visual
Blind/low vision users use screen readers (NVDA, JAWS, VoiceOver). Need proper markup and text alternatives.
⌨️
Motor
Some users navigate only by keyboard (Tab, Enter, Arrow keys). Never trap keyboard focus.
🔊
Auditory
Deaf users need captions on videos and transcripts for audio content.
🧠
Cognitive
Clear language, consistent layout, readable fonts, and meaningful error messages help everyone.

ARIA Roles & Attributes

ARIA (Accessible Rich Internet Applications) adds meaning to elements that HTML alone can't describe — especially for custom components.

AttributePurposeExample

aria-labelProvides an accessible name when visible text isn't enougharia-label="Close menu"
aria-labelledbyPoints to another element's id as the labelaria-labelledby="modal-title"
aria-describedbyPoints to an element with extra description textaria-describedby="hint-text"
aria-hidden="true"Hides decorative element from screen readersIcons, decorative images
aria-expandedWhether a collapsible is open or closedaria-expanded="false"
aria-requiredMarks field as required (for custom controls)aria-required="true"
aria-invalidSignals validation failurearia-invalid="true"
aria-liveAnnounces dynamic content changes to screen readersaria-live="polite"
aria-currentCurrent item in set (nav, breadcrumb, pagination)aria-current="page"
tabindexControls keyboard tab order. 0=natural, -1=programmatic onlytabindex="0"
role="alert"Live region that interrupts screen readers for urgent infoError messages

Accessibility Best Practices

a11y examples 
<!-- ✅ Alt text for images -->
<img src="chart.png" alt="Bar chart showing 40% sales increase in Q1">
<img src="divider.png" alt="" aria-hidden="true"> <!-- decorative -->

<!-- ✅ Skip nav link (for keyboard users) -->
<a href="#main-content" class="skip-link">Skip to main content</a>

<!-- ✅ Accessible button (keyboard + screen reader) -->
<button
  type="button"
  aria-expanded="false"
  aria-controls="dropdown-menu"
>
  Menu <span aria-hidden="true">▾</span>
</button>

<!-- ✅ Live region for dynamic updates -->
<div role="status" aria-live="polite" aria-atomic="true">
  3 results found
</div>

<!-- ✅ Accessible form error -->
<input
  type="email"
  id="email"
  aria-describedby="email-error"
  aria-invalid="true"
>
<span id="email-error" role="alert">Invalid email address</span>
💡
WCAG 2.1 AA — The industry standard for accessibility. 4 principles: Perceivable, Operable, Understandable, Robust (POUR). Aim for a contrast ratio of at least 4.5:1 for text.
🔍 SEO Basics

SEO Basics in HTML

Search Engine Optimization starts with well-structured HTML. Google crawls your HTML before anything else. Get this right and you're already ahead.

On-Page SEO Checklist

ElementBest PracticeImpact
<title>50-60 chars, include target keyword. Unique per page.⭐⭐⭐ Very high
<meta description>150-160 chars, compelling summary, include keyword.⭐⭐ CTR (not ranking)
<h1>One per page, matches user intent, includes primary keyword.⭐⭐⭐ Very high
Heading hierarchyh1 → h2 → h3. Never skip levels. Logical structure.⭐⭐ Medium
alt attributeDescriptive alt text on images. Include keyword naturally.⭐⭐ Image SEO
Semantic HTMLUse proper semantic tags. Google understands your layout.⭐⭐ Medium
Canonical URL<link rel="canonical"> prevents duplicate content.⭐⭐ Technical
Page speedLazy load images, minify HTML, defer JS.⭐⭐⭐ Core Web Vitals
Structured DataJSON-LD schema markup for rich snippets.⭐⭐⭐ Rich results
Internal linkingUse descriptive anchor text, not "click here".⭐⭐ Crawlability

SEO-Optimized HTML Head

SEO Head Tags

    content="Learn MERN stack from scratch. Build 3 real projects with MongoDB, Express, React and Node.js. Free tutorial with source code.">

  <meta property="og:title" content="Best MERN Stack Tutorial">
  <meta property="og:description" content="Learn MERN stack from scratch...">

  {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": "Best MERN Stack Tutorial",
    "author": { "@type": "Person", "name": "RyzeN" },

🚀
Core Web Vitals: Google's ranking signals — LCP (Largest Contentful Paint <2.5s), FID/INP (interaction <200ms), CLS (layout shift <0.1). Lazy load images with loading="lazy" and defer scripts to improve all three.
🎬 Audio & Video

Audio & Video

HTML5 introduced native <audio> and <video> elements — no more Flash or plugin requirements. Multiple source formats ensure cross-browser compatibility.

Video Element

HTML5 Video 
<video
  width="720"
  height="405"
  controls            <!-- shows play/pause/volume UI -->
  autoplay           <!-- plays on load (needs muted for Chrome) -->
  muted              <!-- required for autoplay to work -->
  loop               <!-- loops when done -->
  preload="metadata" <!-- none | metadata | auto -->
  poster="thumbnail.jpg" <!-- shown before play -->
  playsinline        <!-- iOS: play inline instead of fullscreen -->
>
  <!-- Multiple formats for browser compatibility -->
  <source src="video.mp4"  type="video/mp4">
  <source src="video.webm" type="video/webm">
  <source src="video.ogv"  type="video/ogg">

  <!-- Subtitles / Captions -->
  <track
    src="captions-en.vtt"
    kind="captions"
    srclang="en"
    label="English"
    default
  >
  <track src="sub-hi.vtt" kind="subtitles" srclang="hi" label="Hindi">

  <!-- Fallback for ancient browsers -->
  Your browser does not support the video tag.
</video>

Audio Element

HTML5 Audio 
<audio
  controls
  loop
  preload="auto"
>
  <source src="audio.mp3"  type="audio/mpeg">
  <source src="audio.ogg"  type="audio/ogg">
  <source src="audio.wav"  type="audio/wav">
  Your browser does not support the audio element.
</audio>

<!-- JS Media API -->
<script>
  const video = document.querySelector('video');
  video.play();           // play
  video.pause();          // pause
  video.currentTime = 0;  // seek to start
  video.volume = 0.5;     // 50% volume
  video.playbackRate = 2; // 2x speed
  // Events
  video.addEventListener('ended', () => console.log('Done!'));
  video.addEventListener('timeupdate', () => console.log(video.currentTime));
</script>

🏷️ Meta Tags

Meta Tags — Complete Reference

Meta tags live in <head> and give browsers, search engines, and social platforms information about your page. They are invisible to users but critical to everything else.

Essential Meta Tags

All Core Meta Tags 
<head>
  <!── MUST-HAVE ─────────────────────────────────────────────────────────>

  <!-- Character encoding. ALWAYS first -->
  <meta charset="UTF-8">

  <!-- Responsive design. ALWAYS second -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- IE compatibility (legacy, rarely needed) -->
  <meta http-equiv="X-UA-Compatible" content="IE=edge">

  <!── SEO META ───────────────────────────────────────────────────────────>

  <!-- Page title (in <title> tag, not meta) -->
  <title>Page Title — Site Name</title>

  <!-- Search engine description snippet -->
  <meta name="description" content="150-160 chars. Compelling summary of this page.">

  <!-- Keywords (low SEO value now, still used by some engines) -->
  <meta name="keywords" content="keyword1, keyword2, keyword3">

  <!-- Author -->
  <meta name="author" content="Zaheer Shaik">

  <!-- Crawler instructions -->
  <meta name="robots" content="index, follow">
  <!-- Options: noindex, nofollow, noarchive, nosnippet -->

  <!-- Canonical URL -->
  <link rel="canonical" href="https://example.com/this-page">

  <!── OPEN GRAPH (og:) — Facebook, LinkedIn, WhatsApp previews ──────────>

  <meta property="og:site_name"   content="RyzeN Dev">
  <meta property="og:type"        content="website">  <!-- or: article, video, profile -->
  <meta property="og:title"       content="Page Title">
  <meta property="og:description" content="Page description">
  <meta property="og:image"       content="https://example.com/og.jpg">
  <!-- og:image should be 1200×630px -->
  <meta property="og:image:width"  content="1200">
  <meta property="og:image:height" content="630">
  <meta property="og:url"         content="https://example.com/this-page">
  <meta property="og:locale"      content="en_US">

  <!── TWITTER CARD —— Twitter/X previews ─────────────────────────────────>

  <meta name="twitter:card"        content="summary_large_image">
  <!-- summary | summary_large_image | app | player -->
  <meta name="twitter:site"        content="@iamzaheershaik">
  <meta name="twitter:creator"     content="@iamzaheershaik">
  <meta name="twitter:title"       content="Page Title">
  <meta name="twitter:description"  content="Page description">
  <meta name="twitter:image"        content="https://example.com/twitter.jpg">

  <!── PWA / MOBILE APP META ───────────────────────────────────────────────>

  <meta name="theme-color" content="#3b82f6">     <!-- browser chrome color -->
  <meta name="mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-title" content="RyzeN">
  <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
  <link rel="manifest" href="/manifest.json">

  <!── ICONS ───────────────────────────────────────────────────────────────>

  <link rel="icon" href="/favicon.ico">
  <link rel="icon" type="image/png" sizes="32x32" href="/favicon-32.png">
  <link rel="apple-touch-icon" href="/apple-icon-180.png">

  <!── PERFORMANCE ─────────────────────────────────────────────────────────>

  <!-- Preconnect to external origins you'll need -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <!-- Preload critical assets -->
  <link rel="preload" href="hero.jpg" as="image">
  <link rel="preload" href="main.js"  as="script">
  <!-- DNS Prefetch -->
  <link rel="dns-prefetch" href="//api.example.com">

  <!── HTTP-EQUIV (browser behavior) ─────────────────────────────────────>

  <!-- Refresh page after 30 seconds -->
  <meta http-equiv="refresh" content="30">
  <!-- Redirect after 3 seconds -->
  <meta http-equiv="refresh" content="3; url=https://newsite.com">
  <!-- Content Security Policy -->
  <meta http-equiv="Content-Security-Policy" content="default-src 'self'">

</head>
