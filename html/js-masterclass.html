Variables
The building blocks of JavaScript — storing and managing data.
📖 Definition

A variable is a named container that stores a value in memory. JavaScript provides three keywords to declare variables: var, let, and const. Each has different scoping rules and behaviors that fundamentally affect how your code works.

🔬 Deep Explanation

var is function-scoped and hoisted to the top of its function. It can be re-declared and re-assigned. Before ES6, this was the only way to declare variables.

let is block-scoped (within {} braces). It cannot be re-declared in the same scope but can be re-assigned. It exists in a "temporal dead zone" from the start of the block until the declaration.

const is also block-scoped. It cannot be re-declared or re-assigned. However, for objects and arrays, the contents can still be mutated — only the binding is immutable.

⌨️ Syntax & Code Examples
javascript — Variable Declarations
// var — function scoped, hoisted
var name = "Alice";
var name = "Bob"; // ✅ re-declaration allowed
// let — block scoped
let age = 25;
age = 26; // ✅ re-assignment allowed
// let age = 30; ❌ SyntaxError: already declared
// const — block scoped, immutable binding
const PI = 3.14159;
// PI = 3.14; ❌ TypeError: Assignment to constant
// ⚠️ const with objects — contents ARE mutable!
const user = { name: "Alice" };
user.name = "Bob"; // ✅ Works! Object mutated
// user = {}; ❌ TypeError: can't reassign const
// Block scoping demo
if (true) {
let x = 10;
var y = 20;
}
// console.log(x); ❌ ReferenceError
console.log(y); // ✅ 20 — var leaks out!
🌍 Real-World Use Cases
const for config
API URLs, environment constants, fixed values that should never change.
let for loops
Loop counters, iterators, values that change during execution.
const for objects
State objects, DOM references, imported modules — prevent accidental reassignment.
❌ Common Mistakes
  • Using var in loops — causes closure issues in async callbacks
  • Assuming const makes objects/arrays immutable (it doesn't)
  • Accessing let/const before declaration (temporal dead zone)
  • Not understanding that var is function-scoped, not block-scoped
  • Forgetting that undeclared variables become global (no strict mode)
✅ Best Practices
  • Default to const — only use let when reassignment is needed
  • Never use var in modern JavaScript
  • Always use "use strict" to catch undeclared variables
  • Use Object.freeze() if you truly need immutable objects
  • Declare variables at the top of their scope for readability
🎯 Interview Tip: The #1 most asked question is the difference between var, let, and const. Know scoping, hoisting, TDZ, and the closure-in-loop gotcha cold. Be ready to explain with a setTimeout inside a for loop example.

Functions
First-class citizens — the heart of JavaScript.
📖 Definition

A function is a reusable block of code designed to perform a task. In JavaScript, functions are first-class objects — they can be assigned to variables, passed as arguments, returned from other functions, and have their own properties.

🔬 Deep Explanation

JavaScript supports multiple function syntaxes: function declarations (hoisted), function expressions (not hoisted), and arrow functions (lexical this). Arrow functions don't have their own this, arguments, or super — they inherit from the enclosing scope.

javascript — Function Types
// 1. Function Declaration (hoisted)
function greet(name) {
return `Hello, ${name}!`;
}
// 2. Function Expression (NOT hoisted)
const greet = function(name) {
return `Hello, ${name}!`;
};
// 3. Arrow Function (lexical this)
const greet = (name) => `Hello, ${name}!`;
// 4. IIFE — Immediately Invoked
(function() {
console.log("Runs immediately!");
})();
// 5. Default Parameters
function createUser(name, role = "viewer") {
return { name, role };
}
// 6. Higher-Order Function
function multiplier(factor) {
return (num) => num * factor;
}
const double = multiplier(2);
console.log(double(5)); // 10
⚠️ Arrow vs Regular: Arrow functions do NOT have their own this. Never use arrows as object methods or constructors. Use regular functions when you need dynamic this binding.
  • Use arrow functions for callbacks and short operations
  • Use named function declarations for top-level logic (better stack traces)
  • Always use default parameters instead of || fallbacks
  • Keep functions small — single responsibility principle
  • Use pure functions when possible (no side effects)
  • Using arrow functions as object methods (loses this context)
  • Forgetting return in multi-line arrow functions with {}
  • Not understanding function hoisting vs expression hoisting
  • Creating deeply nested callbacks instead of using composition

Arrays
Ordered collections — the Swiss army knife of data.
📖 Definition & Deep Explanation

An array is an ordered, indexed collection of values. JavaScript arrays are dynamic — they can grow, shrink, and hold mixed types. They inherit from Array.prototype which provides dozens of built-in methods. Arrays are reference types — assigning an array to a new variable creates a reference, not a copy.

javascript — Array Methods
const nums = [1, 2, 3, 4, 5];
// map — transform each element
const doubled = nums.map(n => n * 2); // [2,4,6,8,10]
// filter — keep matching elements
const evens = nums.filter(n => n % 2 === 0); // [2,4]
// reduce — accumulate to single value
const sum = nums.reduce((acc, n) => acc + n, 0); // 15
// find / findIndex
const found = nums.find(n => n > 3); // 4
// some / every
nums.some(n => n > 4); // true
nums.every(n => n > 0); // true
// flat / flatMap
const nested = [[1,2],[3,[4]]];
nested.flat(Infinity); // [1,2,3,4]
// Chaining methods
const result = nums
.filter(n => n % 2 !== 0)
.map(n => n ** 2)
.reduce((a, b) => a + b, 0); // 35
💡 Mutating vs Non-mutating: push, pop, splice, sort, reverse mutate the original array. map, filter, reduce, slice, concat return new arrays. In interviews, always clarify which you're using.
  • Prefer map/filter/reduce over for loops for data transformations
  • Use Array.from() or spread [...iterable] to convert iterables
  • Use structuredClone() or spread for shallow copies
  • Use Array.isArray() to check if something is an array

Objects
Key-value pairs — the foundation of everything in JS.
📖 Definition & Deep Explanation

Objects are unordered collections of key-value pairs. Almost everything in JavaScript is an object (arrays, functions, dates). Objects can have methods (functions as values), getters/setters, and inherit from prototypes. The prototype chain is how JavaScript implements inheritance.

javascript — Object Operations
// Object creation
const user = {
name: "Alice",
age: 28,
greet() { return `Hi, I'm ${this.name}`; }
};
// Access & modify
user.name; // "Alice" (dot notation)
user["age"]; // 28 (bracket notation)
user.email = "a@b.c"; // add new property
// Object methods
Object.keys(user); // ["name","age","greet","email"]
Object.values(user); // ["Alice", 28, ƒ, "a@b.c"]
Object.entries(user); // [["name","Alice"], ...]
// Spread & merge
const updated = { ...user, age: 29 };
// Optional chaining
user?.address?.city; // undefined (no error)
// Computed property names
const key = "role";
const obj = { [key]: "admin" }; // { role: "admin" }
🎯 Interview Must-Know: Understand this in objects, prototype chain, Object.create(), Object.assign() vs spread, deep vs shallow copy, and property descriptors (Object.defineProperty).

DOM Manipulation
Interact with the browser — modify pages dynamically.
📖 Definition & Deep Explanation

The DOM (Document Object Model) is a tree-structured representation of your HTML page. JavaScript can read, modify, add, or remove elements from this tree. The browser re-renders the page whenever the DOM changes. Understanding DOM manipulation is essential for building interactive UIs without frameworks.

javascript — DOM Methods
// Selecting elements
const el = document.getElementById("app");
const btn = document.querySelector(".submit-btn");
const items = document.querySelectorAll(".item");
// Creating & appending
const div = document.createElement("div");
div.textContent = "Hello World";
div.classList.add("card");
div.setAttribute("data-id", "1");
document.body.appendChild(div);
// innerHTML vs textContent
el.innerHTML = "<strong>Bold</strong>"; // parses HTML
el.textContent = "Plain text"; // safe, no parsing
// Modifying styles
el.style.backgroundColor = "#1a1a2e";
el.classList.toggle("active");
// Removing elements
el.remove();
parent.removeChild(child);
// Document Fragment (batch DOM updates)
const frag = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
const li = document.createElement("li");
li.textContent = `Item ${i}`;
frag.appendChild(li);
}
list.appendChild(frag); // single reflow!
🚨 XSS Warning: Never use innerHTML with user input. Always use textContent or sanitize HTML to prevent cross-site scripting attacks.

Events
Reacting to user actions — clicks, keys, scrolls, and more.
📖 Definition & Deep Explanation

Events are signals that something has happened — a click, keypress, scroll, or form submission. JavaScript uses an event-driven model. Events propagate through the DOM in three phases: capture (top-down), target, and bubble (bottom-up). Understanding this flow is critical for event delegation and preventing unwanted behavior.

javascript — Event Handling
// addEventListener (recommended)
btn.addEventListener("click", (e) => {
console.log("Clicked!", e.target);
});
// Event Delegation — listen on parent
document.getElementById("list")
.addEventListener("click", (e) => {
if (e.target.matches(".delete-btn")) {
e.target.closest(".item").remove();
}
});
// Prevent default behavior
form.addEventListener("submit", (e) => {
e.preventDefault();
// handle form data with JS
});
// Stop propagation
child.addEventListener("click", (e) => {
e.stopPropagation(); // won't bubble up
});
// Custom Events
const event = new CustomEvent("userLogin", {
detail: { userId: 42 }
});
document.dispatchEvent(event);

ES6+ Concepts
Modern JavaScript — the features that changed everything.
📖 Definition & Explanation

ES6 (ECMAScript 2015) introduced revolutionary features: let/const, arrow functions, template literals, classes, Symbols, iterators, generators, for...of, Map/Set, WeakMap/WeakSet, default parameters, and enhanced object literals. Subsequent versions added optional chaining (?.), nullish coalescing (??), Array.prototype.at(), top-level await, and more.

javascript — ES6+ Features
// Template Literals
const msg = `Hello ${name}, you are ${age} years old`;
// Enhanced Object Literals
const obj = { name, age, greet() { return "hi"; } };
// Symbol — unique identifier
const id = Symbol("id");
// Map & Set
const map = new Map([["key", "value"]]);
const set = new Set([1,2,3,1]); // {1,2,3}
// for...of loop
for (const item of [1,2,3]) { console.log(item); }
// Optional Chaining & Nullish Coalescing
const city = user?.address?.city ?? "Unknown";
// Classes
class Animal {
constructor(name) { this.name = name; }
speak() { return `${this.name} makes a sound`; }
}
class Dog extends Animal {
speak() { return `${this.name} barks`; }
}
🎯Interview Focus: Know Map vs Object, Set vs Array, Symbol use cases, iterator/generator protocol, and the difference between for...in and for...of.

Destructuring
Unpack values from arrays and properties from objects — elegantly.
📖 Definition & Explanation

Destructuring is a syntax that lets you extract values from arrays or properties from objects into distinct variables. It works in variable declarations, function parameters, and assignments. Combined with default values and renaming, it makes code dramatically more readable.

javascript — Destructuring Patterns
// Object destructuring
const { name, age, role = "user" } = user;
// Renaming
const { name: userName, age: userAge } = user;
// Nested destructuring
const { address: { city, zip } } = user;
// Array destructuring
const [first, second, ...rest] = [1,2,3,4,5];
// first=1, second=2, rest=[3,4,5]
// Skip elements
const [, , third] = [1,2,3]; // third = 3
// Swap variables
let a = 1, b = 2;
[a, b] = [b, a]; // a=2, b=1
// In function parameters
function createUser({ name, age, role = "viewer" }) {
return { name, age, role };
}
  • Destructure function parameters for cleaner API signatures
  • Use default values to avoid undefined errors
  • Combine with rest operator for flexible functions
  • Use renaming when variable names would conflict

Spread & Rest Operators
Three dots that do two very different things.
📖 Definition & Explanation

The spread operator (...) expands an iterable into individual elements. The rest operator (...) collects multiple elements into an array. Same syntax, opposite behavior — spread unpacks, rest packs. Context determines which: in function params and destructuring it's rest; everywhere else it's spread.

javascript — Spread & Rest
// SPREAD — expanding
const arr1 = [1,2], arr2 = [3,4];
const merged = [...arr1, ...arr2]; // [1,2,3,4]
const copy = [...arr1]; // shallow copy
// Spread in objects
const defaults = { theme: "dark", lang: "en" };
const config = { ...defaults, lang: "fr" };
// { theme: "dark", lang: "fr" }
// Spread in function calls
Math.max(...[5,2,8,1]); // 8
// REST — collecting
function sum(...nums) {
return nums.reduce((a, b) => a + b, 0);
}
sum(1,2,3,4); // 10
// Rest in destructuring
const { name, ...others } = { name: "Alice", age: 25, role: "dev" };
// others = { age: 25, role: "dev" }

Modules
Organize code into reusable, maintainable files.
📖 Definition & Explanation

JavaScript Modules (ESM) let you split code into separate files with their own scope. export makes things available; import brings them in. Each module has its own scope — no global pollution. Modules are strict mode by default, loaded asynchronously, and evaluated once (singleton). CommonJS (require/module.exports) is Node.js's older system; ESM is the modern standard.

javascript — ES Modules
// Named exports (math.js)
export const PI = 3.14159;
export function add(a, b) { return a + b; }
// Default export (logger.js)
export default class Logger { ... }
// Importing
import { add, PI } from './math.js';
import Logger from './logger.js';
import * as math from './math.js';
// Dynamic import (code splitting)
const module = await import('./heavy-module.js');
// Re-export
export { add } from './math.js';

Closures
Functions that remember — the most powerful concept in JS.
📖 Definition & Explanation

A closure is a function that has access to variables from its outer (enclosing) function's scope, even after the outer function has returned. Every function in JavaScript creates a closure. This happens because functions carry a reference to their lexical environment. Closures enable data privacy, factory functions, partial application, and module patterns.

javascript — Closures
// Basic closure — counter
function createCounter() {
let count = 0; // private variable
return {
increment() { return ++count; },
decrement() { return --count; },
getCount() { return count; }
};
}
const counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2
// count is NOT accessible from outside!
// Classic interview gotcha — var in loop
for (var i = 0; i < 3; i++) {
setTimeout(() => console.log(i), 100);
}
// Output: 3, 3, 3 (var is function-scoped!)
// Fix with let (block-scoped)
for (let i = 0; i < 3; i++) {
setTimeout(() => console.log(i), 100);
}
// Output: 0, 1, 2 ✅
// Fix with IIFE closure
for (var i = 0; i < 3; i++) {
((j) => {
setTimeout(() => console.log(j), 100);
})(i);
}

Hoisting
Why JavaScript lets you use things before declaring them.
📖 Definition & Explanation

Hoisting is JavaScript's behavior of moving declarations to the top of their scope during the compilation phase. var declarations are hoisted and initialized to undefined. let/const are hoisted but NOT initialized — they're in a "Temporal Dead Zone" (TDZ) until the declaration line. Function declarations are fully hoisted (body and all). Function expressions are NOT hoisted.

javascript — Hoisting Behavior
console.log(a); // undefined (var hoisted)
var a = 10;
// console.log(b); ❌ ReferenceError (TDZ)
let b = 20;
// Function declaration — fully hoisted
greet(); // ✅ "Hello!" — works before declaration
function greet() { console.log("Hello!"); }
// Function expression — NOT hoisted
// sayBye(); ❌ TypeError: not a function
var sayBye = function() { console.log("Bye!"); };

Callbacks
Passing functions as arguments — async JavaScript begins here.
📖 Definition & Explanation

A callback is a function passed as an argument to another function, to be executed later. Callbacks are the foundation of asynchronous JavaScript. They're used in event handlers, timers, array methods, and Node.js APIs. The main problem with callbacks is "callback hell" — deeply nested callbacks that become unreadable and hard to debug. This led to Promises and async/await.

javascript — Callbacks
// Synchronous callback
function process(arr, callback) {
return arr.map(callback);
}
process([1,2,3], x => x * 2); // [2,4,6]
// Asynchronous callback
setTimeout(() => console.log("After 1s"), 1000);
// ❌ Callback Hell
getUser(id, (user) => {
getPosts(user.id, (posts) => {
getComments(posts[0].id, (comments) => {
console.log(comments); // 😰 pyramid of doom
});
});
});
// Error-first callback pattern (Node.js)
fs.readFile("data.json", (err, data) => {
if (err) return console.error(err);
console.log(data);
});

Promises
The modern solution to async JavaScript — no more callback hell.
📖 Definition & Explanation

A Promise represents a value that may not be available yet but will be resolved in the future (or rejected with an error). Promises have three states: pending, fulfilled, and rejected. They enable chaining with .then(), error handling with .catch(), and cleanup with .finally(). Static methods like Promise.all(), Promise.race(), Promise.allSettled(), and Promise.any() handle multiple promises.

javascript — Promises
// Creating a Promise
const promise = new Promise((resolve, reject) => {
setTimeout(() => resolve("Done!"), 1000);
});
// Consuming
promise
.then(result => console.log(result))
.catch(err => console.error(err))
.finally(() => console.log("Cleanup"));
// Chaining — no nesting!
getUser(id)
.then(user => getPosts(user.id))
.then(posts => getComments(posts[0].id))
.then(comments => console.log(comments))
.catch(err => console.error(err));
// Static methods
Promise.all([p1, p2, p3]); // all must resolve
Promise.allSettled([p1, p2, p3]); // wait for all
Promise.race([p1, p2, p3]); // first to settle
Promise.any([p1, p2, p3]); // first to resolve

Async / Await
Writing asynchronous code that looks and behaves synchronously.
📖 Definition & Explanation

Async/await is syntactic sugar built on top of Promises. Declaring a function with async ensures it always returns a Promise. Inside an async function, the await keyword pauses execution until the awaited Promise settles, non-blocking the main thread. Error handling is handled naturally using standard `try...catch` blocks.

javascript — Async / Await
// Standard Async/Await usage
async function fetchUserData(userId) {
try {
const response = await fetch(`https://api.example.com/users/${userId}`);
if (!response.ok) throw new Error("Network response was not ok");
const data = await response.json();
return data;
} catch (error) {
console.error("Fetching failed:", error);
throw error;
}
}
// Executing parallel tasks (do NOT await sequentially if independent!)
async function loadDashboard() {
// Starts requests in parallel
const userPromise = fetchUserData(1);
const postsPromise = fetch("/posts").then(r => r.json());
// Awaits both together
const [user, posts] = await Promise.all([userPromise, postsPromise]);
console.log(user, posts);
}

The Event Loop
The heartbeat of JavaScript's non-blocking concurrency model.
📖 Definition & Explanation

JavaScript is single-threaded. The Event Loop orchestrates the execution of synchronous code, microtasks, and macrotasks. The Call Stack runs execution contexts. When async tasks (like timers or IO) finish, callbacks queue up. Microtasks (Promises, `queueMicrotask`, `MutationObserver`) have higher priority and empty completely before the next Macrotask (`setTimeout`, `setInterval`, UI rendering) starts.

javascript — Event Loop Execution Order
console.log("1. Start (Sync)");
setTimeout(() => console.log("5. Timeout (Macrotask)"), 0);
Promise.resolve().then(() => console.log("3. Promise 1 (Microtask)"));
queueMicrotask(() => console.log("4. queueMicrotask (Microtask)"));
console.log("2. End (Sync)");
// Output: 1 -> 2 -> 3 -> 4 -> 5
🎯Interview Focus: Event loop priority output questions are guaranteed in Elite JS rounds. Always trace synchronous code -> entire Microtask queue -> single Macrotask -> render -> repeat.

Debouncing
Delaying execution until a period of inactivity passes.
📖 Definition & Explanation

Debouncing ensures a high-frequency function is only executed after a specified delay of user inactivity. If the event triggers again before the delay lapses, the timer resets. Ideal for search autocompletes and window resizing.

javascript — Debounce Implementation
function debounce(func, delay) {
let timerId;
return function (...args) {
clearTimeout(timerId);
timerId = setTimeout(() => func.apply(this, args), delay);
};
}
// Usage
const handleSearch = debounce((q) => console.log("Fetching:", q), 300);

Throttling
Guaranteeing a consistent rate of execution over time.
📖 Definition & Explanation

Throttling limits the rate at which a function can fire. Unlike debouncing, throttling guarantees execution at regular intervals as long as triggers continue. Highly essential for scroll tracking, shooting games, and button spam prevention.

javascript — Throttle Implementation
function throttle(func, limit) {
let inThrottle;
return function (...args) {
if (!inThrottle) {
func.apply(this, args);
inThrottle = true;
setTimeout(() => inThrottle = false, limit);
}
};
}
